## Reply variants

1. If they ask "why not just use ChatGPT/Cursor?"

HAICHI is not trying to replace them. It is for the local-agent part: roles, plans, handoffs, retries, and evidence in one local workspace instead of scattered tabs and scripts.

2. If they ask "does it need paid APIs?"

No. Personal can run with local Ollama. Pro adds optional providers and broader workflows, but the first workflow should work locally.

3. If they ask "is this SaaS?"

No. HAICHI v1.1 is a local Windows/Linux app. The website is for download, docs, and checkout, not a hosted account portal.

4. If they ask "what should I test first?"

Use a real code change and run Reviewer -> Verifier. Ask Reviewer for risks/missing tests, then Verifier to challenge each claim and keep only the top three fixes.

5. If they say "I do not need another dashboard"

Fair. The product only makes sense if your current local-agent work is already scattered. If one chat is enough, HAICHI is probably unnecessary.

6. If they ask "what does Pro add?"

Pro is the separate licensed build for optional cloud providers, unlimited agents/workflows, and private Tailscale remote access.

7. If they ask about privacy

The default path is local. Provider keys stay in a local ignored credential file. File access is workspace-scoped and opt-in per model/agent.

8. If install fails

Please send the platform, install step, and exact error without secrets. That is more useful than a general bug report.

9. If they want the founder price

Try Personal first. If the workflow is relevant, open the feedback link and ask for a founder code before buying Pro. Do not buy the normal €49 checkout if you are expecting the €29 test price.

10. If they say "looks too early"

It is early. The current test is whether the Reviewer -> Verifier workflow is useful enough to justify polishing the product further.
